# 🏦 SQL Real-World Project – Banking Fraud Detection & Financial Analytics

A comprehensive, production-style **Banking Fraud Detection & Financial Analytics SQL project** designed to demonstrate **expert-level SQL skills** across:

- Data Engineering  
- Data Science  
- AI / Machine Learning Systems  
- Backend & API Development  
- Business Intelligence & Risk Analytics  

This repository simulates a real-world **banking transactions system** with fraud detection logic, risk scoring, financial KPIs, and ML-ready feature engineering using SQL.

---

## 🔥 Overview

Modern banking systems process millions of transactions daily.  
Fraud detection, financial monitoring, and risk analysis rely heavily on **optimized SQL pipelines** before data reaches dashboards or machine learning models.

This project demonstrates how SQL is used to:

- Monitor real-time financial activity  
- Detect suspicious transactions  
- Generate fraud risk scores  
- Build ML-ready datasets  
- Power executive dashboards  
- Support backend financial APIs  

It bridges the gap between:

- Basic transaction queries  
- Advanced analytical SQL  
- Enterprise-grade fraud detection logic  
- ML feature engineering in SQL  

---

## 📂 Repository Structure

### 1️⃣ Dataset (Simulated Banking System)

- `data/banking_transactions.csv` → 10,000 transactions  
- Columns include:
  - `Transaction_ID`
  - `Customer_ID`
  - `Transaction_Date`
  - `Transaction_Type`
  - `Amount`
  - `Merchant_Category`
  - `City`
  - `Payment_Method`
  - `Is_Fraud`

Dataset spans **30 days of transactional activity**, suitable for structured daily SQL practice.

---

### 2️⃣ Core SQL Foundations (Financial Context)

- Transaction summaries  
- Revenue calculations  
- Fraud case filtering  
- Customer activity tracking  
- Aggregations with `GROUP BY` & `HAVING`  
- Data constraints & integrity checks  

---

### 3️⃣ Fraud Detection Analytics

- Fraud rate calculations  
- Fraud distribution by:
  - Payment method  
  - City  
  - Merchant category  
  - Amount bucket  
- High-risk customer identification  
- Rapid transaction detection (same-day patterns)  
- Revenue loss due to fraud  

---

### 4️⃣ Window Functions & Risk Modeling

- `RANK()` for customer risk ranking  
- `ROW_NUMBER()` for transaction ordering  
- `LAG()` for detecting unusual spending spikes  
- Running revenue totals  
- 7-day moving averages  
- `NTILE()` for customer segmentation
